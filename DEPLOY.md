# Deploying Task Manager

This project is ready to deploy from the `my_task_manager` folder. The Django settings module is `tasker.settings`.

## Recommended: PythonAnywhere

PythonAnywhere is the easiest option for this project because you can keep SQLite and do not need to manage a server from PowerShell.

1. Create a PythonAnywhere account.
2. Upload this `my_task_manager` folder, or push it to GitHub and clone it on PythonAnywhere.
3. Open a Bash console on PythonAnywhere and go to the folder that contains `manage.py`.
4. Create a virtualenv with Python 3.12 or newer.
5. Install dependencies:

```bash
pip install -r requirements.txt
```

6. Run database migrations:

```bash
python manage.py migrate
```

7. Collect static files:

```bash
python manage.py collectstatic
```

8. In the PythonAnywhere Web tab, create a Manual Configuration web app.
9. Set the virtualenv to the one you created.
10. Set the source code / working directory to the folder that contains `manage.py`.
11. Edit the PythonAnywhere WSGI file. Use this shape, replacing `yourusername`:

```python
import os
import sys

path = '/home/yourusername/my_task_manager'
if path not in sys.path:
    sys.path.insert(0, path)

os.environ['DJANGO_SETTINGS_MODULE'] = 'tasker.settings'
os.environ['DJANGO_DEBUG'] = 'False'
os.environ['DJANGO_SECRET_KEY'] = 'replace-this-with-a-long-random-secret'
os.environ['DJANGO_ALLOWED_HOSTS'] = 'yourusername.pythonanywhere.com'
os.environ['DJANGO_CSRF_TRUSTED_ORIGINS'] = 'https://yourusername.pythonanywhere.com'
os.environ['DJANGO_SESSION_COOKIE_SECURE'] = 'True'
os.environ['DJANGO_CSRF_COOKIE_SECURE'] = 'True'

from django.core.wsgi import get_wsgi_application
application = get_wsgi_application()
```

12. In the PythonAnywhere Web tab, add static files:

```text
URL: /static/
Directory: /home/yourusername/my_task_manager/staticfiles
```

13. If you use uploaded files/photos, add media files:

```text
URL: /media/
Directory: /home/yourusername/my_task_manager/media
```

14. Press Reload in the Web tab. Your site should open at:

```text
https://yourusername.pythonanywhere.com
```

## Optional: Render

Render works too. This project now includes `render.yaml`, so the easiest path is a Render Blueprint. The Blueprint creates both the Django web service and a PostgreSQL database.

1. Push the `my_task_manager` project to GitHub.
2. Open Render.
3. Go to Blueprints.
4. Create a new Blueprint Instance.
5. Connect the GitHub repository.
6. Select the repository branch that contains `render.yaml`.
7. Apply the Blueprint and wait for the build to finish.

Render will use:

```bash
bash build.sh
```

as the build command, and:

```bash
gunicorn tasker.wsgi:application
```

as the start command.

The Render app receives these important environment variables from `render.yaml`:

```text
DJANGO_DEBUG=False
DJANGO_SECRET_KEY=<generated by Render>
DATABASE_URL=<from the Render PostgreSQL database>
DJANGO_SESSION_COOKIE_SECURE=True
DJANGO_CSRF_COOKIE_SECURE=True
```

`RENDER_EXTERNAL_HOSTNAME` is added to `ALLOWED_HOSTS` automatically in `tasker/settings.py`, so you do not need to know the final `.onrender.com` URL before the first deploy.
